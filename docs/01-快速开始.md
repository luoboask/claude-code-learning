## 1. Claude Code CLI 是什么？

**Claude Code CLI** 是一个面向软件工程任务的终端 Agent：

```
┌─────────────────────────────────────────┐
│  用户："帮我修复这个 bug"                 │
├─────────────────────────────────────────┤
│  Claude Code:                           │
│  1. 读取文件分析问题                     │
│  2. 定位 bug 位置                        │
│  3. 应用修复                             │
│  4. 运行测试验证                         │
│  5. 提交更改                             │
└─────────────────────────────────────────┘
```

### 核心能力

| 能力 | 说明 |
|------|------|
| **读写文件** | 读取、编辑、创建文件 |
| **执行 Shell** | 运行命令、脚本、测试 |
| **搜索代码** | 全局搜索、正则匹配 |
| **调用工具** | MCP、LSP、Web 搜索 |
| **运行子代理** |  fork 子任务、并行处理 |
| **持久记忆** | 跨会话记忆、用户偏好 |

## 3. 核心概念

### 3.1 查询循环 (Query Loop)

Claude Code 的核心是一个无限循环：

```
while (true) {
  1. 调用 Claude API (流式)
  2. 检测工具调用
  3. 执行工具 (权限检查 → 执行 → 结果)
  4. 结果回灌到模型
  5. 继续循环或完成
}
```

### 3.2 工具系统

45 个工具分为 8 类：

```
核心工具：Bash, FileRead, FileWrite, FileEdit, Glob, Grep
Web 工具：WebFetch, WebSearch
代理工具：AgentTool, SkillTool
任务工具：TaskCreate, TaskUpdate, TaskList, ...
集成工具：MCPTool, LSPTool
配置工具：ConfigTool, PlanMode, Worktree
通信工具：AskUserQuestion, SendMessage
条件工具：Sleep, Cron, PowerShell, REPL
```

### 3.3 记忆系统

4 层记忆类型：

```
User Memory     → 用户角色、偏好、知识
Feedback Memory → 工作方式指导 (规则 + Why + How)
Project Memory  → 项目目标、截止日期、决策
Reference Memory → 外部资源位置 (Linear/Grafana/Slack)
```

### 3.4 权限系统

三层规则 + 四种模式：

```
规则:
- alwaysAllowRules  (总是允许)
- alwaysDenyRules   (总是拒绝)
- alwaysAskRules    (总是询问)

模式:
- default  (默认模式)
- bypass   (绕过模式)
- auto     (自动模式)
- plan     (计划模式)
```
