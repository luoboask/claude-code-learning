## 1. 入门问题

### Q1: Claude Code CLI 是什么？

**A**: Claude Code CLI 是一个面向软件工程任务的终端 Agent：
- 能读写文件
- 能执行 Shell 命令
- 能搜索代码库
- 能调用工具/MCP/LSP
- 能运行子代理和技能

## 2. 架构问题

### Q4: Claude Code 的核心组件有哪些？

**A**: 核心组件包括：
- **QueryEngine**: 会话引擎
- **query 循环**: 查询编排
- **Tool 系统**: 45 个工具实现
- **State 管理**: 状态存储
- **Memory 系统**: 持久化记忆
- **Ink 渲染**: 终端 UI

---

### Q5: 查询循环是如何工作的？

**A**: 查询循环流程：
1. 调用 Claude API（流式）
2. 检测工具调用
3. 执行工具（权限检查 → 执行 → 结果）
4. 结果回灌到模型
5. 继续循环或完成

---

### Q6: 状态管理是如何实现的？

**A**: 使用最小状态管理原语：
```typescript
function createStore<T>(initialState: T) {
  let state = initialState
  const listeners = new Set<() => void>()
  
  return {
    getState: () => state,
    setState: (updater) => {
      const next = updater(state)
      if (Object.is(next, state)) return
      state = next
      listeners.forEach(fn => fn())
    },
    subscribe: (listener) => {
      listeners.add(listener)
      return () => listeners.delete(listener)
    },
  }
}
```

## 3. 记忆系统问题

### Q7: 记忆系统的作用是什么？

**A**: 记忆系统提供跨会话的持久化知识存储：
- 记录用户角色、偏好、知识
- 记录工作方式指导
- 记录项目目标和决策
- 记录外部资源位置

---

### Q8: 4 层记忆类型有什么区别？

**A**:
| 类型 | 用途 | 作用域 |
|------|------|--------|
| **User** | 用户角色、偏好 | 私有 |
| **Feedback** | 工作方式指导 | 私有/团队 |
| **Project** | 项目目标、决策 | 私有/团队 |
| **Reference** | 外部资源位置 | 团队 |

---

### Q9: 如何保存记忆？

**A**: 两步流程：
1. 将记忆写入独立文件（如 `user_role.md`）
2. 在 `MEMORY.md` 中添加指针

```markdown
# Step 1: 创建文件
---
name: 用户角色
description: 用户是数据科学家
type: user
---

用户是数据科学家，专注于可观测性。

# Step 2: 更新 MEMORY.md
- [用户角色](user_role.md) — 数据科学家，专注可观测性
```

---

### Q10: 记忆的大小限制是多少？

**A**: 
- 行数限制：200 行
- 字节限制：25KB
- 双重限制，先按行截断，再按字节截断

## 4. 工具系统问题

### Q11: 如何定义一个工具？

**A**: 使用 buildTool() 构建器：
```typescript
export const MyTool = buildTool({
  name: 'MyTool',
  description: 'Tool description',
  inputSchema: z.object({
    param: z.string(),
  }),
  async call(args, context) {
    // 实现逻辑
    return { data: { result: 'success' } }
  },
  isReadOnly: () => true,
  isConcurrencySafe: () => true,
})
```

---

### Q12: 工具如何执行？

**A**: 执行流程：
1. 分区（并发安全批 vs 串行批）
2. 并发执行读操作
3. 串行执行写操作
4. 权限检查
5. 执行工具
6. 返回结果

---

### Q13: 权限系统是如何工作的？

**A**: 三层规则 + 四种模式：
```
规则:
- alwaysAllowRules  → 总是允许
- alwaysDenyRules   → 总是拒绝
- alwaysAskRules    → 总是询问

模式:
- default  → 默认模式
- bypass   → 绕过模式
- auto     → 自动模式
- plan     → 计划模式
```

## 6. 集成问题

**A**: 
```json
{

      "command": "npx",

    }
  }

```

### Q17: MCP 支持哪些传输方式？

**A**: 4 种传输方式：
- **stdio**: 本地进程
- **sse**: HTTP SSE

### Q20: 如何测试权限规则？

**A**: 
```typescript
// 单元测试
describe('Permission System', () => {
  it('should allow git status', async () => {
    const result = await hasPermissionsToUseTool(BashTool, { command: 'git status' }, context)
    expect(result.behavior).toBe('allow')
  })
  
  it('should deny rm -rf /', async () => {
    const result = await hasPermissionsToUseTool(BashTool, { command: 'rm -rf /' }, context)
    expect(result.behavior).toBe('deny')
  })
})
```
