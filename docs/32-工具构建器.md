## 1. 为什么需要构建器？

### 1.1 问题背景

**没有构建器时**:
```typescript
// 手动实现所有字段，容易遗漏
export const MyTool: Tool = {
  name: 'MyTool',
  inputSchema: z.object({...}),
  call: async (...) => {...},
  isEnabled: () => true,
  isConcurrencySafe: () => false,
  isReadOnly: () => false,
  isDestructive: () => false,
  checkPermissions: (...) => Promise.resolve({...}),
  // ... 40+ 字段，容易遗漏
}
```

**问题**:
- ❌ 需要手动实现 40+ 字段
- ❌ 容易遗漏默认值
- ❌ 代码重复
- ❌ 维护困难

### 1.2 构建器解决方案

```typescript
// 使用 buildTool()
export const MyTool = buildTool({
  name: 'MyTool',
  inputSchema: z.object({...}),
  call: async (...) => {...},
  // 其他字段自动填充默认值
})
```

**优点**:
- ✅ 只需定义必要字段
- ✅ 自动填充默认值
- ✅ 类型安全
- ✅ 易于维护

## 2. 默认值定义

### 2.1 完整默认值

```typescript
// source/src/Tool.ts

const TOOL_DEFAULTS = {
  // ═══════════════════════════════════════════════════════════════
  // 特性检查（保守默认）
  // ═══════════════════════════════════════════════════════════════
  
  isEnabled: () => true,
  isConcurrencySafe: (_input?: unknown) => false,  // 保守：假设不安全
  isReadOnly: (_input?: unknown) => false,         // 保守：假设写入
  isDestructive: (_input?: unknown) => false,
  
  // ═══════════════════════════════════════════════════════════════
  // 权限检查（交给全局系统）
  // ═══════════════════════════════════════════════════════════════
  
  checkPermissions: (
    input: { [key: string]: unknown },
    _ctx?: ToolUseContext,
  ): Promise<PermissionResult> =>
    Promise.resolve({ behavior: 'allow', updatedInput: input }),
  
  // ═══════════════════════════════════════════════════════════════
  // 分类器（跳过）
  // ═══════════════════════════════════════════════════════════════
  
  toAutoClassifierInput: (_input?: unknown) => '',
  
  // ═══════════════════════════════════════════════════════════════
  // UI 渲染（使用名称）
  // ═══════════════════════════════════════════════════════════════
  
  userFacingName: (_input?: unknown) => '',
}
```

### 2.2 默认值设计哲学

| 默认值 | 值 | 原因 |
|--------|-----|------|
| `isEnabled` | `true` | 默认启用，除非明确禁用 |
| `isConcurrencySafe` | `false` | **保守**：假设不安全，避免并发问题 |
| `isReadOnly` | `false` | **保守**：假设写入，需要显式标记只读 |
| `isDestructive` | `false` | **保守**：假设不破坏，需要显式标记 |
| `checkPermissions` | `allow` | 交给全局权限系统处理 |
| `toAutoClassifierInput` | `''` | 跳过分类器，除非显式实现 |

## 3. 构建器函数

### 3.1 完整实现

```typescript
// source/src/Tool.ts

/**
 * 可默认化的工具方法键
 */
type DefaultableToolKeys =
  | 'isEnabled'
  | 'isConcurrencySafe'
  | 'isReadOnly'
  | 'isDestructive'
  | 'checkPermissions'
  | 'toAutoClassifierInput'
  | 'userFacingName'

/**
 * 构建器函数
 */
export function buildTool<D extends AnyToolDef>(def: D): BuiltTool<D> {
  return {
    ...TOOL_DEFAULTS,
    userFacingName: () => def.name,  // 默认使用工具名称
    ...def,
  } as BuiltTool<D>
}
```

### 3.2 类型推导

```typescript
// 类型级 spread， mirroring runtime `{...TOOL_DEFAULTS, ...def}`
type BuiltTool<D> = Omit<D, DefaultableToolKeys> & {
  [K in DefaultableToolKeys]-?: K extends keyof D
    ? undefined extends D[K]
      ? ToolDefaults[K]
      : D[K]
    : ToolDefaults[K]
}
```

## 4. 使用示例

### 4.1 BashTool

```typescript
// source/src/tools/BashTool/BashTool.tsx

export const BashTool = buildTool({
  // ─────────────────────────────────────────────────────────────────
  // 必填字段
  // ─────────────────────────────────────────────────────────────────
  
  name: 'Bash',
  description: 'Execute shell commands',
  
  inputSchema: z.object({
    command: z.string().describe('The command to execute'),
    timeout: z.number().optional().default(30000),
  }),
  
  // ─────────────────────────────────────────────────────────────────
  // 核心方法
  // ─────────────────────────────────────────────────────────────────
  
  async call(args, context, canUseTool, parentMessage, onProgress) {
    // 1. 安全检查
    const securityResult = validateBashCommand(args.command)
    if (securityResult.behavior === 'deny') {
      return { data: { error: securityResult.message } }
    }
    
    // 2. 执行命令
    const result = await exec(args.command, {
      timeout: args.timeout,
      cwd: context.options.cwd,
    })
    
    // 3. 返回结果
    return {
      data: {
        stdout: result.stdout,
        stderr: result.stderr,
        exitCode: result.exitCode,
      },
    }
  },
  
  // ─────────────────────────────────────────────────────────────────
  // 可选字段（覆盖默认值）
  // ─────────────────────────────────────────────────────────────────
  
  isConcurrencySafe(input) {
    // 读操作可并发
    return /^(ls|cat|head|tail|grep|find)\s/.test(input.command)
  },
  
  isReadOnly(input) {
    // 检查是否是只读命令
    return /^(ls|cat|head|tail|grep|find|wc)\s/.test(input.command)
  },
  
  isDestructive(input) {
    // 检查是否是破坏性命令
    return /^(rm|mv|cp|chmod|chown)\s/.test(input.command)
  },
})
```

// 自定义 WeatherTool

import { buildTool } from './Tool'
import { z } from 'zod/v4'

export const WeatherTool = buildTool({
  name: 'Weather',
  description: 'Get weather forecast for a location',
  searchHint: 'weather forecast temperature',
  
  inputSchema: z.object({
    location: z.string().describe('City name or coordinates'),
    days: z.number().optional().default(3),
  }),
  
  async call(args, context) {
    const response = await fetch(
      `https://wttr.in/${args.location}?format=j1`

    const data = await response.json()
    
    return {
      data: {
        location: data.nearest_area[0].areaName[0].value,
        forecast: data.weather,
      },
    }
  },
  
  // 覆盖默认值
  isReadOnly: () => true,
  isConcurrencySafe: () => true,
  
  // UI 渲染
  renderToolResultMessage(content, options) {
    return (
      <Box flexDirection="column">
        <Text bold>Weather for {content.location}:</Text>
        {content.forecast.map(day => (
          <Text key={day.date}>
            {day.date}: {day.avgTemp}°C
          </Text>
        ))}
      </Box>
    )
  },
})

## 5. 最佳实践

### 5.1 何时覆盖默认值

```markdown
✅ 应该覆盖:

1. isConcurrencySafe
   - 读操作：返回 true
   - 写操作：返回 false

2. isReadOnly
   - 只读操作：返回 true
   - 写入操作：返回 false

3. isDestructive
   - 破坏性操作：返回 true
   - 非破坏性：返回 false

4. checkPermissions
   - 工具特定权限逻辑
   - 危险路径检查

❌ 不应该覆盖:

1. isEnabled (除非需要条件启用)
2. toAutoClassifierInput (除非需要分类)
3. userFacingName (默认使用 name 即可)
```

### 5.2 保守默认值

```typescript
// ✅ 保守的自定义默认值
isConcurrencySafe: () => false,  // 默认不安全
isReadOnly: () => false,         // 默认写入
isDestructive: () => false,      // 默认不破坏

// ❌ 激进的默认值（不要这样做）
isConcurrencySafe: () => true,   // 可能导致并发问题
isReadOnly: () => true,          // 可能错误标记写操作
```

### 5.3 错误处理

```typescript
// ✅ 完整的错误处理
async call(args, context) {
  try {
    // 验证输入
    const validation = await this.validateInput?.(args, context)
    if (validation && !validation.result) {
      return { data: { error: validation.message } }
    }
    
    // 执行逻辑
    const result = await execute(args)
    
    return { data: result }
  } catch (error) {
    return {
      data: {
        error: error instanceof Error ? error.message : 'Unknown error',
      },
    }
  }
}

// ❌ 不完整的错误处理
async call(args, context) {
  // 没有错误处理
  return await execute(args)
}
```

## 6. 常见问题

### Q1: 为什么默认值是保守的？

**A**: 安全优先。保守默认值确保：
- 默认不并发（避免竞态条件）
- 默认写入（需要显式标记只读）
- 默认不破坏（需要显式标记破坏性）

### Q2: 可以不使用 buildTool 吗？

**A**: 可以，但不推荐。直接实现 Tool 接口需要：
- 手动实现所有 40+ 字段
- 容易遗漏默认值
- 维护困难

### Q3: 如何添加新的默认值？

**A**: 修改 TOOL_DEFAULTS：
```typescript
const TOOL_DEFAULTS = {
  // ... 现有默认值
  newField: () => defaultValue,
}

// 同时更新类型
type DefaultableToolKeys = [
  // ... 现有键
  'newField',
]
```
