## 1. 权限决策流程

### 1.1 完整流程图

```
工具调用请求
    │
    ↓
┌─────────────────────────────────┐
│ 1. hasPermissionsToUseTool()     │
│ - 检查 alwaysAllowRules          │
│ - 检查 alwaysDenyRules           │
│ - 检查 alwaysAskRules            │
└───────────────┬─────────────────┘
                │
          ┌─────┴─────┐
          │           │
      [匹配规则]   [未匹配]
          │           │
          ↓           ↓
    返回决策    ┌─────────────────┐
                │ 2. 检查权限模式  │
                │ - bypass         │
                │ - auto           │
                │ - default        │
                └────────┬────────┘
                         │
                   ┌─────┴─────┐
                   │           │
               [允许]     [询问]
                   │           │
                   ↓           ↓
            执行工具   ┌─────────────────┐
                       │ 3. 交互对话框   │
                       │ - 用户确认      │
                       │ - 保存规则      │
                       └────────┬────────┘
                                │
                                ↓
                         ┌──────────────┐
                         │ 4. 追踪决策   │
                         │ - 记录计数    │
                         │ - 触发 fallback│
                         └──────────────┘
```

### 1.2 核心函数

```typescript
// source/src/utils/permissions/permissions.ts

export async function hasPermissionsToUseTool(
  tool: Tool,
  input: Record<string, unknown>,
  toolUseContext: ToolUseContext,
  assistantMessage: AssistantMessage,
  toolUseID: string,
): Promise<PermissionResult> {
  const { toolPermissionContext } = toolUseContext.getAppState()
  
  // 1. 检查 alwaysAllowRules
  const allowRule = matchRule(toolPermissionContext.alwaysAllowRules, tool, input)
  if (allowRule) {
    return { behavior: 'allow', decisionReason: { type: 'rule', rule: allowRule } }
  }
  
  // 2. 检查 alwaysDenyRules
  const denyRule = matchRule(toolPermissionContext.alwaysDenyRules, tool, input)
  if (denyRule) {
    return { behavior: 'deny', decisionReason: { type: 'rule', rule: denyRule } }
  }
  
  // 3. 检查 alwaysAskRules
  const askRule = matchRule(toolPermissionContext.alwaysAskRules, tool, input)
  if (askRule) {
    return { behavior: 'ask', decisionReason: { type: 'rule', rule: askRule } }
  }
  
  // 4. 检查权限模式
  switch (toolPermissionContext.mode) {
    case 'bypass':
      if (!isDangerousTool(tool, input)) {
        return { behavior: 'allow', decisionReason: { type: 'mode', mode: 'bypass' } }
      }
      break
    
    case 'auto':
      if (isAutoModeAllowed(tool, input)) {
        return { behavior: 'allow', decisionReason: { type: 'mode', mode: 'auto' } }
      }
      break
  }
  
  // 5. 默认：询问用户
  return { behavior: 'ask' }
}
```

```typescript
// source/src/utils/permissions/permissionRuleParser.ts

export function permissionRuleValueFromString(ruleString: string): PermissionRuleValue {
  // 解析规则字符串
  // 'git *' → { type: 'glob', pattern: 'git *' }
  // '/src/*' → { type: 'path', pattern: '/src/*' }
  
  if (ruleString.includes('*')) {
    return { type: 'glob', pattern: ruleString }
  }
  
  if (ruleString.startsWith('/')) {
    return { type: 'path', pattern: ruleString }
  }
  
  return { type: 'exact', value: ruleString }
}

export function ruleMatches(ruleValue: PermissionRuleValue, input: Record<string, unknown>): boolean {
  switch (ruleValue.type) {
    case 'exact':
      return input.command === ruleValue.value
    
    case 'glob':
      return picomatch(ruleValue.pattern)(input.command as string)
    
    case 'path':
      const path = input.path as string
      return picomatch(ruleValue.pattern)(path)
    
    default:
      return false
  }
}
```

## 3. 交互式权限对话框

### 3.1 对话框组件

```typescript
// source/src/components/permissions/PermissionRequest.tsx

export function PermissionRequest({
  tool,
  input,
  description,
  onAllow,
  onDeny,
  onAlwaysAllow,
  onAlwaysDeny,
}: PermissionRequestProps): React.ReactNode {
  const [selected, setSelected] = useState<'allow' | 'deny'>('allow')
  
  useInput((input, key) => {
    if (key.leftArrow || key.rightArrow) {
      setSelected(prev => prev === 'allow' ? 'deny' : 'allow')
    } else if (key.name === 'a') {
      onAlwaysAllow(selected === 'allow')
    } else if (key.name === 'd') {
      onAlwaysDeny(selected === 'allow')
    } else if (key.return) {
      if (selected === 'allow') {
        onAllow()
      } else {
        onDeny()
      }
    }
  })
  
  return (
    <Box flexDirection="column" border={true} padding={1}>
      <Text bold>Permission Request</Text>
      <Text>{tool.name}</Text>
      <Text dimColor>{description}</Text>
      
      <Box marginTop={1}>
        <Text color={selected === 'allow' ? 'green' : undefined}>
          [A] Allow
        </Text>
        <Text marginX={2}>|</Text>
        <Text color={selected === 'deny' ? 'red' : undefined}>
          [D] Deny
        </Text>
      </Box>
      
      <Box marginTop={1}>
        <Text dimColor>
          [Shift+A] Always Allow  [Shift+D] Always Deny
        </Text>
      </Box>
    </Box>
  )
}
```

### 3.2 用户选项

| 选项 | 快捷键 | 效果 |
|------|--------|------|
| **Allow** | A | 允许本次执行 |
| **Deny** | D | 拒绝本次执行 |
| **Always Allow** | Shift+A | 添加到 alwaysAllowRules |

## 4. 权限追踪

### 4.1 拒绝计数

```typescript
// source/src/utils/permissions/denialTracking.ts

export type DenialTrackingState = {
  count: number
  limit: number
  fallbackToPrompting: boolean
}

export const DENIAL_LIMITS = {
  autoMode: 3,  // 自动模式 3 次拒绝后 fallback
}

export function recordDenial(state: DenialTrackingState): DenialTrackingState {
  return {
    ...state,
    count: state.count + 1,
    fallbackToPrompting: state.count >= DENIAL_LIMITS.autoMode,
  }
}

export function recordSuccess(state: DenialTrackingState): DenialTrackingState {
  return {
    ...state,
    count: 0,  // 重置计数
  }
}

export function shouldFallbackToPrompting(state: DenialTrackingState): boolean {
  return state.fallbackToPrompting
}
```

### 5.2 模式切换函数

```typescript
// source/src/utils/permissions/getNextPermissionMode.ts

export function getNextPermissionMode(
  currentMode: PermissionMode,
): PermissionMode {
  switch (currentMode) {
    case 'default':
      return 'bypass'
    case 'bypass':
      return 'auto'
    case 'auto':
      return 'default'
    case 'plan':
      return 'default'
  }
}
```

### 5.3 模式说明

| 模式 | 说明 | 使用场景 |
|------|------|----------|
| **default** | 按规则检查 | 正常使用 |
| **bypass** | 自动允许 (危险工具除外) | 受信任环境 |
| **auto** | 自动允许 (需用户事先同意) | 自动化任务 |
| **plan** | 计划模式特殊处理 | 计划模式 |

## 6. 最佳实践

### 6.1 配置权限规则

```markdown
✅ 应该做:

1. 从宽松开始
   - 先配置 alwaysAllowRules
   - 逐步添加限制

2. 使用具体规则
   - 'git status' 而非 'git *'
   - '/tmp/*' 而非 '*'

3. 定期审查规则
   - 删除过时规则
   - 合并重复规则

❌ 不应该做:

1. 过度限制
   - '*' 拒绝所有
   - 影响正常工作

2. 规则冲突
   - 同一命令在多个规则中
```

---

### 6.2 安全考虑

```markdown
✅ 应该做:

1. 危险命令总是询问
   - rm, mv, cp 等
   - curl, wget 等网络命令

2. 路径限制
   - 限制写入目录
   - 禁止写入敏感路径

3. 审计日志
   - 记录所有权限决策
   - 监控异常模式

❌ 不应该做:

1. 在生产环境使用 bypass 模式
2. 允许写入系统目录
3. 忽视权限警告
```
