## 1. 三层权限规则

### 1.1 规则定义

```typescript
// source/src/Tool.ts

export type ToolPermissionContext = DeepImmutable<{
  mode: PermissionMode
  
  alwaysAllowRules: ToolPermissionRulesBySource
  alwaysDenyRules: ToolPermissionRulesBySource
  alwaysAskRules: ToolPermissionRulesBySource
  
  // ...
}>
```

### 1.2 规则说明

| 规则 | 用途 | 示例 |
|------|------|------|
| **alwaysAllowRules** | 总是允许 | `git status`, `ls`, `cat` |
| **alwaysDenyRules** | 总是拒绝 | `rm -rf /`, `curl evil.com` |
| **alwaysAskRules** | 总是询问 | 文件写入、命令执行 |

### 1.3 规则格式

```typescript
// 按工具名称组织
type ToolPermissionRulesBySource = {
  command?: Record<string, string[]>  // Bash 命令模式
  file?: Record<string, string[]>     // 文件路径模式
  // ... 其他工具
}

// 示例
{
  command: {
    'Bash': ['git *', 'ls *', 'cat *'],  // alwaysAllow
    'Bash': ['rm *', 'curl *'],          // alwaysDeny
  },
  file: {
    'FileRead': ['/src/*'],              // alwaysAllow
    'FileWrite': ['/tmp/*'],             // alwaysAllow
  },
}
```

## 2. 四种权限模式

### 2.1 模式定义

```typescript
type PermissionMode = 
  | 'default'  // 默认模式
  | 'bypass'   // 绕过模式
  | 'auto'     // 自动模式
  | 'plan'     // 计划模式
```

### 2.2 模式说明

| 模式 | 说明 | 使用场景 |
|------|------|----------|
| **default** | 按规则检查 | 正常使用 |
| **bypass** | 自动允许 (危险工具除外) | 受信任环境 |
| **auto** | 自动允许 (需用户事先同意) | 自动化任务 |
| **plan** | 计划模式特殊处理 | 计划模式 |

### 2.3 模式切换

```typescript
// 获取下一个权限模式
export function getNextPermissionMode(
  currentMode: PermissionMode,
): PermissionMode {
  switch (currentMode) {
    case 'default':
      return 'bypass'
    case 'bypass':
      return 'auto'
    case 'auto':
      return 'default'
    case 'plan':
      return 'default'
  }
}
```

## 3. 权限决策流程

### 3.1 完整流程

```
工具调用
    │
    ↓
┌─────────────────────────────────┐
│ 1. 检查 alwaysAllowRules         │
│ - 匹配 → allow                  │
└───────────────┬─────────────────┘
                │
                ↓ (未匹配)
┌─────────────────────────────────┐
│ 2. 检查 alwaysDenyRules          │
│ - 匹配 → deny                   │
└───────────────┬─────────────────┘
                │
                ↓ (未匹配)
┌─────────────────────────────────┐
│ 3. 检查 alwaysAskRules           │
│ - 匹配 → ask                    │
└───────────────┬─────────────────┘
                │
                ↓ (未匹配)
┌─────────────────────────────────┐
│ 4. 检查权限模式                  │
│ - bypass → allow (危险除外)     │
│ - auto → allow (需同意)         │
│ - default/plan → ask            │
└───────────────┬─────────────────┘
                │
                ↓
┌─────────────────────────────────┐
│ 5. 默认：询问用户                │
│ - 弹出权限确认对话框             │
└─────────────────────────────────┘
```

### 3.2 实现代码

```typescript
// source/src/utils/permissions/permissions.ts

export async function hasPermissionsToUseTool(
  tool: Tool,
  input: Record<string, unknown>,
  toolUseContext: ToolUseContext,
  assistantMessage: AssistantMessage,
  toolUseID: string,
): Promise<PermissionResult> {
  const { toolPermissionContext } = toolUseContext.getAppState()
  
  // 1. 检查 alwaysAllowRules
  const allowRule = matchRule(toolPermissionContext.alwaysAllowRules, tool, input)
  if (allowRule) {
    return { behavior: 'allow', decisionReason: { type: 'rule', rule: allowRule } }
  }
  
  // 2. 检查 alwaysDenyRules
  const denyRule = matchRule(toolPermissionContext.alwaysDenyRules, tool, input)
  if (denyRule) {
    return { behavior: 'deny', decisionReason: { type: 'rule', rule: denyRule } }
  }
  
  // 3. 检查 alwaysAskRules
  const askRule = matchRule(toolPermissionContext.alwaysAskRules, tool, input)
  if (askRule) {
    return { behavior: 'ask', decisionReason: { type: 'rule', rule: askRule } }
  }
  
  // 4. 检查权限模式
  switch (toolPermissionContext.mode) {
    case 'bypass':
      if (!isDangerousTool(tool, input)) {
        return { behavior: 'allow', decisionReason: { type: 'mode', mode: 'bypass' } }
      }
      break
    
    case 'auto':
      if (isAutoModeAllowed(tool, input)) {
        return { behavior: 'allow', decisionReason: { type: 'mode', mode: 'auto' } }
      }
      break
  }
  
  // 5. 默认：询问用户
  return { behavior: 'ask' }
}
```

## 4. 规则匹配

### 4.1 matchRule 函数

```typescript
// source/src/utils/permissions/permissions.ts

function matchRule(
  rules: ToolPermissionRulesBySource,
  tool: Tool,
  input: Record<string, unknown>,
): PermissionRule | null {
  for (const source of PERMISSION_RULE_SOURCES) {
    const sourceRules = rules[source] || {}
    const toolRules = sourceRules[tool.name] || []
    
    for (const ruleString of toolRules) {
      const ruleValue = permissionRuleValueFromString(ruleString)
      if (ruleMatches(ruleValue, input)) {
        return {
          source,
          ruleBehavior: 'allow',  // 或 'deny' / 'ask'
          ruleValue,
        }
      }
    }
  }
  
  return null
}
```

### 4.2 规则匹配示例

```typescript
// Bash 命令规则匹配
const rules = {
  command: {
    'Bash': ['git *', 'ls *', 'cat *'],
  },
}

// 匹配 'git status'
ruleMatches('git *', { command: 'git status' })  // true

// 匹配 'ls -la'
ruleMatches('ls *', { command: 'ls -la' })  // true

// 不匹配 'rm file'
ruleMatches('git *', { command: 'rm file' })  // false
```

```
┌─────────────────────────────────────────┐
│  权限请求                                │
```

## 6. 权限追踪

### 6.1 拒绝追踪

```typescript
// source/src/utils/permissions/denialTracking.ts

export type DenialTrackingState = {
  count: number
  limit: number
  fallbackToPrompting: boolean
}

export const DENIAL_LIMITS = {
  autoMode: 3,  // 自动模式 3 次拒绝后 fallback
}

export function recordDenial(state: DenialTrackingState): DenialTrackingState {
  return {
    ...state,
    count: state.count + 1,
    fallbackToPrompting: state.count >= DENIAL_LIMITS.autoMode,
  }
}
```

### 6.2 成功追踪

```typescript
export function recordSuccess(state: DenialTrackingState): DenialTrackingState {
  return {
    ...state,
    count: 0,  // 重置计数
  }
}
```

## 7. 最佳实践

### 7.1 配置权限规则

```markdown
✅ 应该做:

1. 从宽松开始
   - 先配置 alwaysAllowRules
   - 逐步添加限制

2. 使用具体规则
   - 'git status' 而非 'git *'
   - '/tmp/*' 而非 '*'

3. 定期审查规则
   - 删除过时规则
   - 合并重复规则

❌ 不应该做:

1. 过度限制
   - '*' 拒绝所有
   - 影响正常工作

2. 规则冲突
   - 同一命令在多个规则中
```

### 7.2 选择权限模式

```markdown
default 模式:
✅ 正常使用
✅ 安全优先
❌ 需要频繁确认

bypass 模式:
✅ 受信任环境
✅ 减少确认
❌ 仍有危险工具检查

auto 模式:
✅ 自动化任务
✅ 无需确认
❌ 需用户事先同意
```

## 8. 常见问题

### Q1: 如何添加权限规则？

**A**: 
```json
// settings.json
{
  "permissionRules": {
    "alwaysAllow": {
      "command": {
        "Bash": ["git status", "ls *", "cat *"]
      }
    }
  }
}
```

### Q2: 如何禁用权限检查？

**A**: 
```bash
# 使用 bypass 模式
/permissions set mode bypass

# 或使用 --dangerously-skip-permissions
claude --dangerously-skip-permissions
```

### Q3: 权限规则优先级？

**A**: 
```
1. alwaysAllowRules (最高)
2. alwaysDenyRules
3. alwaysAskRules
4. 权限模式
5. 默认询问 (最低)
```
