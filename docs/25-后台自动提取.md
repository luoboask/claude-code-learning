## 1. 系统概述

### 1.1 为什么需要自动提取？

**问题**: 手动记录记忆容易遗漏，用户可能忘记要求保存重要信息。

**解决**: 后台代理定期分析对话，自动提取值得保存的记忆。

```
主代理对话
    │
    ↓
┌─────────────────────────────────┐
│ 后台代理定期运行                 │
│ - 分析对话内容                   │
│ - 识别值得保存的信息             │
│ - 自动写入记忆                   │
└─────────────────────────────────┘
    │
    ↓
记忆系统自动更新
```

### 1.2 系统架构

```
┌─────────────────────────────────┐
│ 主代理                           │
│ - 处理用户请求                   │
│ - 执行工具                       │
│ - 可选：手动保存记忆             │
└───────────────┬─────────────────┘
                │
                │ 会话消息
                ↓
┌─────────────────────────────────┐
│ 后台代理 (extract-memories)     │
│ - 定期运行                       │
│ - 分析消息                       │
│ - 提取记忆                       │
│ - 写入记忆文件                   │
└─────────────────────────────────┘
```

### 2.2 自动记忆启用检查

```typescript
// source/src/memdir/paths.ts

/**
 * 是否启用自动记忆功能
 * 默认启用。优先级链 (第一个定义的获胜):
 *   1. CLAUDE_CODE_DISABLE_AUTO_MEMORY 环境变量 (1/true → OFF, 0/false → ON)
 *   2. CLAUDE_CODE_SIMPLE (--bare) → OFF
 *   3. CCR 无持久存储 → OFF (no CLAUDE_CODE_REMOTE_MEMORY_DIR)
 *   4. autoMemoryEnabled in settings.json (支持项目级 opt-out)
 *   5. 默认：启用
 */
export function isAutoMemoryEnabled(): boolean {
  const envVal = process.env.CLAUDE_CODE_DISABLE_AUTO_MEMORY
  if (isEnvTruthy(envVal)) return false
  if (isEnvDefinedFalsy(envVal)) return true
  
  // --bare / SIMPLE
  if (isEnvTruthy(process.env.CLAUDE_CODE_SIMPLE)) return false
  
  // CCR 无持久存储
  if (
    isEnvTruthy(process.env.CLAUDE_CODE_REMOTE) &&
    !process.env.CLAUDE_CODE_REMOTE_MEMORY_DIR
  ) {
    return false
  }
  
  // settings.json 检查
  const settings = getInitialSettings()
  if (settings.autoMemoryEnabled !== undefined) {
    return settings.autoMemoryEnabled
  }
  
  return true
}
```

### 2.3 启用/禁用方式

```bash
# 禁用自动记忆
export CLAUDE_CODE_DISABLE_AUTO_MEMORY=1

# 启用自动记忆
export CLAUDE_CODE_DISABLE_AUTO_MEMORY=0

# 或使用 settings.json
{
  "autoMemoryEnabled": false  // 禁用
}
```

## 3. 触发条件

### 3.1 提取判断逻辑

```typescript
// source/src/services/SessionMemory/sessionMemory.ts

export function shouldExtractMemory(messages: Message[]): boolean {
  // 1. 检查是否满足初始化阈值
  const currentTokenCount = tokenCountWithEstimation(messages)
  if (!isSessionMemoryInitialized()) {
    if (!hasMetInitializationThreshold(currentTokenCount)) {
      return false
    }
    markSessionMemoryInitialized()
  }

  // 2. 检查是否满足更新间最小 token 阈值
  const hasMetTokenThreshold = hasMetUpdateThreshold(currentTokenCount)

  // 3. 检查是否满足工具调用阈值
  const toolCallsSinceLastUpdate = countToolCallsSince(messages, lastMemoryMessageUuid)
  const hasMetToolCallThreshold = toolCallsSinceLastUpdate >= getToolCallsBetweenUpdates()

  // 4. 检查最后助手轮次是否有工具调用（安全提取）
  const hasToolCallsInLastTurn = hasToolCallsInLastAssistantTurn(messages)

  // 5. 触发提取当:
  //    - 两个阈值都满足 (tokens AND tool calls), 或
  //    - 最后轮次无工具调用 AND token 阈值满足 (自然对话断点)
  const shouldExtract =
    (hasMetTokenThreshold && hasMetToolCallThreshold) ||
    (hasMetTokenThreshold && !hasToolCallsInLastTurn)

  if (shouldExtract) {
    const lastMessage = messages[messages.length - 1]
    if (lastMessage?.uuid) {
      lastMemoryMessageUuid = lastMessage.uuid
    }
    return true
  }

  return false
}
```

### 3.2 触发条件详解

| 条件 | 说明 | 阈值 |
|------|------|------|
| **Token 阈值** | 自上次提取后的 token 增长 | ~5000 tokens |
| **工具调用阈值** | 自上次提取后的工具调用次数 | ~5 次 |
| **自然断点** | 最后轮次无工具调用 | - |

### 3.3 触发流程

```
会话进行中
    │
    ↓
┌─────────────────────────────────┐
│ 定期检查是否应该提取             │
│ - Token 增长 ≥ 5000?            │
│ - 工具调用 ≥ 5 次？              │
│ - 最后轮次无工具调用？           │
└───────────────┬─────────────────┘
                │
          ┌─────┴─────┐
          │           │
       [满足]      [不满足]
          │           │
          ↓           ↓
┌──────────────┐  ┌──────────────┐
│ 启动后台代理 │  │ 继续会话     │
│ - 提取记忆   │  │              │
│ - 写入文件   │  │              │
└──────────────┘  └──────────────┘
```

## 4. 与主代理的协作

### 4.1 分工

```
主代理:
- 处理用户请求
- 执行工具
- 可选：手动保存记忆（用户明确要求时）

后台代理:
- 定期分析对话
- 自动提取记忆
- 写入记忆文件
- 不干扰主代理工作
```

### 4.2 避免重复

```typescript

export function hasMemoryWritesSince(messages, sinceUuid): boolean {
  // 检查自主代理写入后是否有新的记忆保存
  // 如果有，后台代理跳过该范围，避免重复
}
```

### 4.3 协作流程

```
主代理会话
    │
    ├─→ 用户明确要求保存记忆
    │       └─→ 主代理立即保存
    │               ↓
    │           后台代理跳过该范围
    │
    └─→ 用户未要求保存
            └─→ 后台代理定期分析
                    ↓
                自动提取值得保存的信息
```

## 5. 提取内容

### 5.1 值得提取的信息

```markdown
✅ 应该提取:

1. 用户细节
   - 角色、偏好、知识
   - 技术栈、经验水平

2. 反馈指导
   - 用户纠正 ("不要...")
   - 用户确认 ("对，正是这样")

3. 项目信息
   - 正在进行的工作
   - 截止日期、负责人
   - 技术决策

4. 外部资源
   - Bug 追踪系统
   - 监控仪表板
   - 文档位置

❌ 不应该提取:

1. 可推导的信息
   - 代码模式、架构
   - Git 历史

2. 临时状态
   - 进行中的工作
   - 当前对话上下文

3. CLAUDE.md 已有的内容
```

### 5.2 提取示例

```markdown
# 对话片段

用户：我在调查我们有哪些日志记录
助手：[分析代码库，找到日志配置]
用户：我是数据科学家，专注于可观测性
助手：[保存用户记忆：数据科学家，专注可观测性]

用户：不要在这些测试中 mock 数据库
助手：[保存反馈记忆：集成测试禁止 mock 数据库]

用户：我们周四后冻结合并，移动端要发布
助手：[保存项目记忆：2026-03-05 后冻结合并]

# 后台代理提取

分析对话后，自动提取：
1. 用户角色：数据科学家
2. 反馈：测试禁止 mock
3. 项目：合并冻结

写入记忆文件：
- user_role.md
- feedback_testing.md
- project_freeze.md
```

## 8. 最佳实践

### 8.1 配置建议

```markdown
✅ 推荐配置:

1. 默认启用自动记忆
   - 无需手动记录
   - 不会遗漏重要信息

2. 简单模式禁用
   - --bare 模式下不需要自动记忆
   - 减少开销

3. 远程会话配置持久存储
   - CLAUDE_CODE_REMOTE_MEMORY_DIR
   - 确保记忆跨会话持久化
```

### 8.2 与手动记录配合

```markdown
手动记录 + 自动提取:

1. 用户明确要求保存 → 主代理立即保存
2. 用户未要求但值得保存 → 后台代理自动提取
3. 避免重复 → 后台代理检查主代理是否已保存

最佳实践:
- 用户明确要求时立即保存
- 后台代理作为补充，捕获遗漏
```

## 9. 常见问题

### Q1: 自动提取会干扰主代理吗？

**A**: 不会。后台代理：
- 独立运行，不阻塞主代理
- 使用 fork 的子代理
- 主代理继续处理用户请求

### Q2: 如何禁用自动提取？

**A**: 
```bash
export CLAUDE_CODE_DISABLE_AUTO_MEMORY=1
```
或在 settings.json 中：
```json
{ "autoMemoryEnabled": false }
```

### Q3: 自动提取的频率？

**A**: 
- 基于 token 增长和工具调用次数
- 通常每 5000 tokens 或 5 次工具调用
- 自然对话断点也会触发

### Q4: 提取的记忆准确吗？

**A**: 
- 后台代理使用与主代理相同的记忆规则
- 提取值得保存的信息
- 用户可以后续更新或删除
