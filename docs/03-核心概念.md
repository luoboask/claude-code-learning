## 1. Agent CLI 是什么？

### 定义

**Agent CLI** = 终端界面 + AI 模型 + 工具系统 + 权限控制

用户通过自然语言与 AI 交互，AI 通过工具执行实际任务。

### 核心价值

| 价值 | 说明 |
|------|------|
| **自然语言编程** | 用说话的方式写代码 |
| **自动化重复任务** | 让 AI 执行繁琐操作 |
| **知识辅助** | AI 知道最佳实践 |
| **安全执行** | 权限控制保护系统 |

### 典型工作流程

```
┌─────────────┐
│ 用户输入     │ "帮我修复这个 bug"
└──────┬──────┘
       │
       ↓
┌─────────────┐
│ AI 理解意图   │ 分析需求，规划步骤
└──────┬──────┘
       │
       ↓
┌─────────────┐
│ 工具调用     │ 读取文件 → 分析问题 → 修改代码
└──────┬──────┘
       │
       ↓
┌─────────────┐
│ 权限检查     │ 用户确认是否允许修改
└──────┬──────┘
       │
       ↓
┌─────────────┐
│ 执行并反馈   │ 显示修改结果
└─────────────┘
```

## 2. 核心术语

### 2.1 查询循环 (Query Loop)

**定义**: Claude Code 的核心执行机制，一个无限循环处理用户请求。

```typescript
// 简化版查询循环
while (true) {
  // 1. 调用 Claude API (流式)
  const stream = callClaudeAPI(messages, systemPrompt)
  
  for await (const event of stream) {
    // 2. 处理流式响应
    if (event.type === 'tool_use') {
      toolUseBlocks.push(event)
    }
    yield event  // 推送给 UI
  }
  
  // 3. 检查是否有工具需要执行
  if (toolUseBlocks.length > 0) {
    const results = yield* runTools(toolUseBlocks)
    messages.push(...toolResults)  // 结果回灌
    continue  // 继续循环
  }
  
  // 4. 完成
  return { type: 'complete' }
}
```

**关键概念**:
- **流式处理**: 边接收边处理，不用等完整响应
- **工具检测**: 从响应中识别 tool_use 块
- **结果回灌**: 工具执行结果作为新消息加入对话
- **迭代机制**: 循环直到任务完成或达到限制

---

### 2.2 工具 (Tool)

**定义**: AI 可以调用的功能单元，如执行命令、读写文件、搜索代码等。

**核心属性**:
```typescript
type Tool = {
  name: string           // 工具名称
  inputSchema: ZodType   // 输入验证 Schema
  call(...): Promise<ToolResult>  // 执行方法
  isReadOnly(input): boolean      // 是否只读
  isConcurrencySafe(input): boolean // 是否可并发
  // ... 40+ 字段
}
```

**工具分类**:
```
核心工具：Bash, FileRead, FileWrite, FileEdit, Glob, Grep
Web 工具：WebFetch, WebSearch
代理工具：AgentTool, SkillTool
任务工具：TaskCreate, TaskUpdate, TaskList, ...
集成工具：MCPTool, LSPTool
配置工具：ConfigTool, PlanMode, Worktree
通信工具：AskUserQuestion, SendMessage
条件工具：Sleep, Cron, PowerShell, REPL
```

---

### 2.3 记忆系统 (Memory System)

**定义**: 持久化存储用户偏好、项目信息、反馈指导的系统。

**4 层记忆类型**:

| 类型 | 用途 | 作用域 | 示例 |
|------|------|--------|------|
| **User** | 用户角色、偏好、知识 | 私有 | "用户是数据科学家" |
| **Feedback** | 工作方式指导 | 私有/团队 | "测试必须用真实数据库" |
| **Project** | 项目目标、决策 | 私有/团队 | "2026-03-05 冻结合并" |
| **Reference** | 外部资源位置 | 团队 | "Linear 项目'INGEST'追踪 bug" |

**记忆结构**:
```markdown
---
name: 记忆名称
description: 一行描述
type: user|feedback|project|reference
---

记忆内容

对于 feedback/project 类型:
**Why:** 原因
**How to apply:** 如何应用
```

---

### 2.4 权限系统 (Permission System)

**定义**: 控制 AI 能否执行特定工具调用的机制。

**三层规则**:
```
alwaysAllowRules  → 总是允许 (如：git status, ls, cat)
alwaysDenyRules   → 总是拒绝 (如：rm -rf /, curl evil.com)
alwaysAskRules    → 总是询问 (如：文件写入、命令执行)
```

**四种模式**:
```
default  → 默认模式，按规则检查
bypass   → 绕过模式，自动允许 (危险工具除外)
auto     → 自动模式，自动允许 (需用户事先同意)
plan     → 计划模式，特殊处理
```

**权限决策流程**:
```
工具调用
    │
    ↓
检查 alwaysAllowRules → allow
检查 alwaysDenyRules  → deny
检查 alwaysAskRules   → ask
检查权限模式          → allow/deny
默认                  → ask (弹出对话框)
```

---

### 2.5 上下文 (Context)

**定义**: 传递给 AI 模型的所有信息，包括对话历史、系统提示、工具定义等。

**上下文组成**:
```
┌─────────────────────────────────┐
│ 系统提示                         │
│ - 工具定义                       │
│ - 记忆指令                       │
│ - 行为指南                       │
├─────────────────────────────────┤
│ 对话历史                         │
│ - 用户消息                       │
│ - AI 回复                        │
│ - 工具结果                       │
├─────────────────────────────────┤
│ 附加上下文                       │
│ - Git 状态                       │
│ - CLAUDE.md 指令                 │
│ - 文件内容                       │
└─────────────────────────────────┘
```

**上下文限制**:
- **Token 限制**: 模型有最大上下文窗口 (如 200K tokens)
- **自动压缩**: 超出限制时自动压缩历史
- **优先级**: 最近的消息优先级更高

---

### 2.6 子代理 (Subagent)

**定义**: 从主代理 fork 出来的独立执行单元，用于并行处理任务。

**使用场景**:
- 长时间运行的任务
- 需要隔离上下文的任务
- 并行处理多个子任务

**子代理类型**:
```
Coordinator Mode (协调器模式)
├─→ Main Agent (主代理)
│   └─→ 协调多个 Worker
└─→ Worker Agents (工作代理)
    ├─→ 执行具体任务
    └─→ 向主代理汇报
```

---

### 2.7 MCP (Model Context Protocol)

**定义**: 一种开放协议，允许 AI 模型与外部系统和工具交互。

**核心概念**:
- **Server**: 提供工具和服务的外部系统
- **Client**: Claude Code 中的 MCP 客户端
- **Transport**: 通信方式 (stdio/sse/websocket/streamable-http)
- **Resource**: 服务器提供的资源
- **Tool**: 服务器提供的可调用工具

**MCP 服务器示例**:
```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/tmp"]
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"]
    }
  }
}
```

---

### 2.8 LSP (Language Server Protocol)

**定义**: 一种开放协议，用于编辑器/IDE 与语言服务器之间的通信。

**LSP 功能**:
- **代码补全**: 智能提示
- **跳转定义**: 跳转到函数/类定义
- **查找引用**: 查找所有引用位置
- **诊断**: 错误和警告
- **重构**: 重命名、提取方法等

**LSP 集成**:
```
Claude Code CLI
    │
    ↓
LSP Client
    │
    ├─→ TypeScript Server
    ├─→ Python Server
    └─→ Go Server
```

```
用户输入
    │
    ↓
┌─────────────────────────────────┐
│ main.tsx                        │
│ - 解析 CLI 参数                   │
│ - 初始化                         │
│ - 启动渲染                       │
└───────────────┬─────────────────┘
                │
                ↓
┌─────────────────────────────────┐
│ App.tsx → REPL.tsx              │
│ - 捕获用户输入                   │
│ - 添加到消息队列                 │
└───────────────┬─────────────────┘
                │
                ↓
┌─────────────────────────────────┐
│ QueryEngine.submitMessage()     │
│ - 包装 canUseTool               │
│ - 构建系统提示                   │
│ - 处理用户输入                   │
└───────────────┬─────────────────┘
                │
                ↓
┌─────────────────────────────────┐
│ query() 生成器                   │
│ - 调用 Claude API               │
│ - 处理流式响应                   │
│ - 检测工具调用                   │
└───────────────┬─────────────────┘
                │
          ┌─────┴─────┐
          │           │
    [流式响应]   [tool_use]
          │           │
          │           ↓
          │   ┌─────────────────┐

          │   │ - 执行工具       │
          │   │ - 返回结果       │
          │   └────────┬────────┘
          │            │
          └────────────┴────────┐
                               │
                               ↓
                    ┌─────────────────────────┐
                    │ 结果回灌到 query 循环     │
                    │ - 构建 tool_result      │

                    └───────────┬─────────────┘
                                │
                                ↓
                    ┌─────────────────────────┐
                    │ Ink 组件渲染输出         │
                    └─────────────────────────┘
```

## 5. 设计模式

### 5.1 生成器模式 (Generator Pattern)

**用途**: 流式处理、查询循环、工具执行

```typescript
async function* query(params): AsyncGenerator<StreamEvent, Terminal> {
  const stream = callClaudeAPI(...)
  for await (const event of stream) {
    yield event  // 推送给 UI
  }
  if (hasToolUse) {
    yield* runTools(...)  // 嵌套生成器
    continue
  }
  return { type: 'complete' }
}
```

**优点**:
- 自然表达异步流
- 支持嵌套生成器 (`yield*`)
- 可随时暂停/恢复

---

### 5.2 建造者模式 (Builder Pattern)

**用途**: 工具构建、消息构建

```typescript
const TOOL_DEFAULTS = {
  isEnabled: () => true,
  isConcurrencySafe: () => false,
  isReadOnly: () => false,
  // ...
}

export function buildTool<D>(def: D): BuiltTool<D> {
  return {
    ...TOOL_DEFAULTS,
    ...def,
  } as BuiltTool<D>
}
```

**优点**:
- 统一接口
- 复用执行逻辑
- 易于测试

---

### 5.3 观察者模式 (Observer Pattern)

**用途**: 状态管理、事件通知

```typescript
function createStore<T>(initialState: T) {
  let state = initialState
  const listeners = new Set<() => void>()

  return {
    getState: () => state,
    setState: (updater) => {
      const next = updater(state)
      if (Object.is(next, state)) return
      state = next
      listeners.forEach(fn => fn())  // 通知所有订阅者
    },
    subscribe: (listener) => {
      listeners.add(listener)
      return () => listeners.delete(listener)
    },
  }
}
```

**优点**:
- 松耦合
- 支持多订阅者
- 自动通知
